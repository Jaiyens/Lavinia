## What went well

1. **Epic 7's architecture discipline.** The canonical surface registry (7.1) became the single validation source for every downstream navigation story — add/remove a surface in one place, get type errors at the call sites. The capability-by-omission seam (7.2) scaffolded empty, then 7.3 dropped `navigate` into the read-safe set exactly as designed. 7.4 hit a genuine contradiction in the planning docs (a part that is `transient: true` AND in `message.parts` AND dedupe-by-id — impossible under AI SDK v6) and resolved it by reading the installed SDK types before coding. Design A (transient + `useChat({ onData })`) made exactly-once structural, with Design B documented as a fallback.
2. **Reuse over rewrite, 8 → 9.** Epic 9's PDF path bolted onto Epic 8's full-data loader, coverage footer, and Reports persistence unchanged. One loader, one footer, one store, one download route. The "full-data law" (uncapped export path, strictly separate from the row-capped chat tools, scoped by `farmId` from `deps`) is a clean structural guarantee that a grower can never export another farm's data.
3. **Sharp adversarial review where story files existed.** Epic 10's rate-limiter review caught a HIGH-severity trust bug (leftmost `x-forwarded-for` is client-spoofable; switched to Vercel's non-spoofable `x-vercel-forwarded-for`), an unbounded-memory sweep that made the key cap a no-op, and a throttle test that passed even when the short-circuit was broken (fixed with a Proxy `prisma` that throws on access). Epic 7's review caught race conditions, case-sensitivity mismatches, and dead-ends.
4. **Product law held throughout.** Gentle surfacing (FR22) was non-negotiable: a server-decided httpOnly-cookie first-run nudge with no hydration flash, Home-only, dismissible, never a modal. SCHEDULED bill-due dates are never shown as billed. Money stays tabular, never a lone screaming number. Voice stayed plain operator English with no em dashes.
5. **Determinism as a guarantee.** Every export/PDF builder is a pure function whose bytes are asserted by unit tests; the offline stub drives every skill, so CI makes zero external calls. Batth-scale (183 meters) is asserted in the export and PDF tests — no silent truncation.

## What fought back

1. **The expensive bugs lived in the offline stub, not the type system** (the Epic 5 lesson, repeated). Epic 7's stub lower-cased user text while `filterMeters` matched case-sensitively, so `ag-4` silently never matched stored `AG-4` and emptied the table; a navigation-looking *question* dead-ended instead of falling through to an answer; a stale `messagesRef` mis-attributed the first action chip. All caught in review, none prevented by the stub validating its own input.
2. **Epics 8 & 9 db tests were written but never run.** The overnight pass only does `next build`, not Vitest-with-Postgres. Two tests in `generate-report.db.test.ts` actually fail on HEAD: (1) asserts a contiguous phrase in a stream chunked into 24-char `text-delta` deltas, and (2) asserts a `GeneratedReport` row that `storeReport` best-effort-catches and never persists in that setup. Documented in `deferred-work.md`, left unfixed "to respect story scope." A failing test nobody ran is a deferred problem, not scope discipline.
3. **The `GeneratedReport` migration is hand-authored, not Prisma-generated.** Deterministic and safe, but the Reports / generate / export features 500 in prod unless someone runs `db:migrate` against the target database on merge.
4. **No story files for 8 & 9 meant no acceptance criteria and no per-story review.** The 8.5 owner-skill capability assertion went stale when 9.3 added `generateReport`, and was caught only in a post-hoc fix commit (`dadeac3`). With story files and an Acceptance Auditor, that is caught in the round.

## Significant discoveries

**🚨 Discovery 1 — the farmer-validation build gate.** The Almond `sprint-status.yaml` and `epics.md` both state the heavy build is gated behind farmer validation (PRD Open Q4 / decision D14): *"how we build once validated, not a greenlight to start before a real grower confirms the data-first / gentle-surfacing thesis."* All four epics shipped. The project's own strategic note frames Almond as *an addition, not the wedge* — growers want their data legible upfront, not an AI operator.

**Decision (Jaiyen, this retro): built ahead, validate before exposing.** The gate still stands. Almond is recorded as **built-but-unvalidated**: merged and demo-able, but NOT to be surfaced to real growers (wide public Tour live, first-run nudge enabled for real owners) until a grower confirms the thesis.

**🚨 Discovery 2 — process drift on Epics 8 & 9.** No story files → no acceptance criteria, no per-story adversarial review; db tests written-not-run (two failing); hand-authored migration pending. Epic 10's rate-limiter is in-memory and per-instance, so **Vercel BotID must be enabled before the generative endpoint is on the wide public Tour** (the AR16 gate, only half-closed by 10.3).

**No Epic 11.** Epic 10 was the last Almond epic. The road ahead is one of: Tool 2 (the purchasing agent, its own BMAD effort) or closing the Tool-1 launch blockers from the Epic 5 retro (C1-C4). Almond should not reach growers until Discovery 1 and the BotID gate are settled.

### Process / quality

- **P1 — Reinstate and harden the Epic 5 verification gate (A1).** Any merge touching auth, routing, data-isolation, OR a generative/cost endpoint gets a live round-trip + integration test before "done." Overnight/agentic builds are exempt from live verification *by nature*, so anything built overnight inherits an explicit "verify-before-expose" debt item rather than being marked `done` outright.
- **P2 — Overnight/agentic builds must still produce story files** (or backfill them) so there are acceptance criteria to audit and a per-story review layer. Epics 8 & 9 are the cautionary case (stale capability assertion caught only post-merge).
- **P3 — Reconcile the tracker inside the same merge.** The overnight branch shipped with its own `sprint-status.yaml` still `backlog`; status was reconciled on main afterward. Make "update sprint-status in the merge" part of the overnight protocol.
- **P4 — Prefer Prisma-generated migrations.** If hand-authoring under agentic constraints, the merge checklist must include "run `db:migrate` on the target DB" (this is C1).
- **P5 — Stubs validate / normalize their input** (carry-forward from Epic 5 A3, still being missed). The Epic 7 case-sensitivity bug was the Epic-5 stub trap again.

### Technical debt (track, not exposure-blocking)

- Stub parser simplifications: lens word hijacks a compound meter-open ("show me the table for pump 3"); greedy `(.+)$` capture swallows trailing qualifiers; constant `data-navigate` part id. Live model unaffected.
- `navigate` cannot resolve a finding / "opportunity" directly — the "Open my biggest opportunity" starter has no direct navigate target (possible Epic 7 enhancement).
- Tour rail "Ask Almond" is a dead control when no demo farm exists (no worse than the existing FAB behavior).
- Per-farm generation throttle counts `empty`/`error` attempts — deliberate cost-bound tradeoff; revisit with a peek-then-commit limiter if owners report nuisance lockouts.
- Rate-limit constants hardcoded (30/min/IP, 10/min/farm); env-tunable later.
- AC3 (exactly-once) / AC5 (live-path emission) in 7.4 lack an executable mock-model test; proven structurally + by typecheck.

## Key takeaways

1. **Story files are the review surface.** Epics 7 & 10 (with story files + 3-layer adversarial review) shipped clean; Epics 8 & 9 (no story files) carry the only failing tests and a stale assertion caught post-merge. Agentic speed without acceptance criteria trades correctness confidence for throughput.
2. **Green build is not green behavior** — the Epic 5 lesson, now extended. The overnight build proved compilation, not behavior. Its strength (determinism, pure builders, an offline stub) is exactly why it could not catch the chunked-stream and persistence bugs that only a *run* db test surfaces.
3. **The verification gate (A1) is the recurring miss.** Two retros in a row, the deferred work is "verify live." Until it is a hard merge gate, it will keep being deferred.
4. **Build-ahead is a legitimate choice, but it creates an exposure debt.** Almond is real, polished, and merged — and it must not reach growers until validation and the critical path close. Record the gate, don't quietly cross it.
