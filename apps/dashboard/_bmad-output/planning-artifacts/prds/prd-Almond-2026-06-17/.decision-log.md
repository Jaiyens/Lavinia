## Research grounding (digest captured)

- **Notion's agent takes write actions but gates every create/update/delete/send behind a confirm-with-preview, and inherits the user's permissions.** Because Terra chose read-only v1 (D3), we sidestep the confirmation-risk surface entirely; we keep only the lightweight "here's what I'm about to make" preview for heavier artifacts.
- Notion model = "AI teammate," assisted autonomy, undo as a first-class affordance, custom-agents = scheduled background workflows (our "future scheduled reports").
- Agent-as-front-door comparables (Glean, Sierra, Decagon) all go chat-first; Terra deliberately does NOT (D2) — lower risk, dashboard stays the hero.
- Export-in-agents pattern: pure-JS libs cover CSV/XLSX/PPTX; Chromium only needed for pixel-faithful HTML→PDF and is the serverless-gotcha path — avoid it.
