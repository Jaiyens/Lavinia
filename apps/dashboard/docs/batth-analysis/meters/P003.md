## (b) RATE SCHEDULE — decoded in plain English

**Stored schedule: `AG5B`. Latest billed tariff on the print: "AG5B Large Time-of-Use Agricultural Power".**

- **What it is.** AG-5 is one of PG&E's **legacy** agricultural time-of-use rate families (it predates
  the current AG-A / AG-B / AG-C lineup). The trailing "B" denotes the **large** size class within the
  family (>35 kW connected load — appropriate for a 1,200 GPM pump). In the rate engine this maps to
  `{ family: "AG-5", sizeClass: "large", realTier: "large", legacy: true }`
  (see `src/lib/energy/rate-lever.ts`, the `planFromLabel` table). `inventory.legacy = true` confirms it.
- **Charge structure (AG-5 large, legacy TOU-demand).** A monthly/daily **customer charge**, **time-of-use
  energy** ($/kWh that differs by peak / partial-peak / off-peak window and by summer/winter season), and a
  **summer peak-period demand charge** ($/kW). AG-5, like the current AG-C, carries a demand charge — unlike
  the AG-A / AG-B "no separately-stated demand" rates. The engine's published peak window for demand is the
  **5–8pm rate peak** (`rates.ts`).
- **Why it is on this rate.** It is a **legacy grandfathered** assignment: a large ag pump enrolled years ago
  on AG-5 and never migrated to the current AG-A/B/C card. Legacy AG-5 meters are exactly the population the
  rate-optimization lever exists to re-examine, because mixed legacy/current rates across 183 meters almost
  guarantee some now sit on the wrong schedule.

All dollar figures below are produced by **deterministic pure functions in `src/lib/energy`** (no AI). The only

numbers in the first place.

### Finding 1 — Rate optimization (legacy AG-5 re-examination) — **SILENT / fail-closed**
- **Engine:** `src/lib/energy/rate-lever.ts` (`planFromLabel` tags AG5B as `legacy:true`; the back-test gate
  `BACK_TEST_BAND_PCT = 5%` decides whether a dollar may be quoted).
- **Arithmetic:** The lever back-tests by re-pricing the meter's **own billed usage** on the dated card and
  checking it lands within ±5% of the printed totals. Here the only billed cycle has **0 kWh and 0 demand**, so
  there is nothing to re-price — the energy/demand basis is empty. The lever therefore **cannot** compute a
  defensible AG-5 → AG-A/B/C switch figure.
- **Why no dollar:** Per the engine's own rule, an off-band / no-usage legacy meter falls back to a **qualitative
  finding only** ("this is a legacy AG-5 meter; worth a rate review once a full-usage cycle is captured"), never a
  number. This is intentional fail-closed behavior, not a miss.
- **Data we HAVE:** legacy flag, schedule, one idle cycle, NEM annual totals.
- **Data we NEED:** **15-minute interval kWh** for at least one full irrigation-season cycle (summaries carry no
  interval kWh). Without intervals, any AG-5 → AG-B/AG-C "savings" the engine could emit is **sign-ambiguous and
  must not be quoted** (this is the documented interval-required caveat).

### Finding 2 — Bill audit (own-bill anomaly) — **NOT TRIGGERABLE this cycle**

- **Arithmetic:** It flags a cycle whose total tops the meter's **same-season median** by more than the tolerance

  cycle**, and it is a $35.83 idle month, so there is no comparator set and nothing to flag.
- **Data we HAVE:** one cycle. **NEED:** ≥3 same-season billed cycles with totals + peaks.
