**Stored schedule: `HAGB`. Latest billed tariff on the print: "AGB Ag35+ kW Med Use" (the AG-B family).**

- **What it is.** The stored code `HAGB` resolves to the current **AG-B** agricultural schedule
  ("Ag 35+ kW, Medium Use"). In the rate engine, AGB maps to
  `{ family: "AG-B", sizeClass: "large", realTier: "large" }` (the `planFromLabel` table in
  `src/lib/energy/rate-lever.ts`); it prices off the card's **AG-B2** row. AG-B is a **current** (non-legacy)
  schedule — `inventory.legacy = null`, and the cycle's `isLegacyTou = false`.
- **Charge structure.** AG-B carries: a **daily customer charge**, **time-of-use energy** ($/kWh by peak /
  off-peak window), and — for the 35+ kW medium-use tier — a **demand charge ($/kW)**. The peak window the engine
  uses for demand is the **5–8pm rate peak** (`rates.ts`). Note this print actually shows **two** demand
  components (a structured Maximum-Demand charge and a separate "Demand Charge 1" line — see section c).
- **Why it is on this rate.** "Ag 35+ kW Medium Use" is the appropriate **current** tier for a mid-size
  agricultural pump (~35–~100 kW connected). It is already on a current rate (not legacy), so the headline
  rate-switch lever has much less room here than on a legacy AG-4/AG-5 meter.
