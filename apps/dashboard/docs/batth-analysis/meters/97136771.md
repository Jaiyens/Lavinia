## (b) RATE SCHEDULE — decoded

- **Stored schedule:** `HAGC` → **HAGC / AG-C at transmission (high) voltage.**
- **Plain English:** `HAGC` is **not a separate tariff** — it is PG&E's billing code for **Schedule AG Rate AG-C taken at *transmission* (high) voltage** (the 'H' prefix = high/transmission voltage). Use the **Transmission-voltage rows** of the AG-C column: **Max Demand $6.45/kW** (much lower than the $21.43/kW secondary) and the **Summer Peak-Demand $29.92/kW** (the same across all voltage classes), with AG-C's customer charge (~$43.58/mo) and cheap energy.
- **Why a meter is on HAGC:** transmission service is only available where the customer is served directly off PG&E's transmission system — typically the **large solar-paired pump banks** (e.g. the 840 kW / 1,092 kW wells in this farm's fleet). Many of this farm's newer high-GPM wells are HAGC.
- **The arbitrage still applies:** even at transmission voltage the **$29.92/kW summer 5-8pm peak-demand charge is unchanged** — so a HAGC meter whose true peak-window demand is small is paying that charge for nothing. Demote-candidate test is identical: pull trailing-12-month interval kW, find the max 15-min demand, and if comfortably <35 kW re-elect to AG-A1/AG-A2.
- **Demand exposure:** high (the 5-8pm peak charge dominates). The transmission voltage only softens the *max-demand* row, not the peak-demand row.
- **Mapping caveat:** this decode is from `brief-pge-ag-rates.md` (Advice 7846-E, eff. 3/1/2026). PG&E re-prices several times a year; re-pull the tariff before billing-grade use. The grower's stored token here is `HAGC`.
